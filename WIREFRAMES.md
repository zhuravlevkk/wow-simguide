# WoW SimGuide - Wireframe Mockups

Три варианта редизайна интерфейса. Каждый решает проблемы текущего дизайна:
линейность, отсутствие навигации, перегруженность хедера, плоская иерархия шагов.

---

## Вариант A: "Wizard / Пошаговый мастер"

### Плюсы варианта A:
- Пользователь не теряется: один шаг = один экран
- Прогресс-бар мотивирует дойти до конца
- Чистый, не перегруженный интерфейс
- Отлично работает на мобильных
- Легко добавлять/убирать шаги

### Минусы:
- Нельзя быстро перескочить к нужному шагу (только через прогресс-бар)
- Опытным пользователям может быть медленно листать

- Ресурсы всегда под рукой в сайдбаре

### Минусы:
- Сложнее в реализации
- На мобильных сайдбар скрыт — теряется часть UX
- Больше визуального шума

- Может казаться перегруженной для новичков
- Нужна реализация scroll-spy + sticky header

## Рекомендация

**Вариант C (Timeline)** — лучший баланс между удобством, визуалом и атмосферой WoW.
Можно комбинировать с элементами Варианта A (прогресс-бар) для усиления UX.

Альтернативно, **Вариант A (Wizard)** идеален если основная аудитория — новички,
которые впервые симят персонажа.
